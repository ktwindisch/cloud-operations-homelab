# ADR 000: Project Vision

## Status

Accepted

## Context

I am building a long-term Cloud and DevOps engineering project using a repurposed Lenovo ThinkPad T530.

This laptop has personal and professional history. It was the machine I used while studying at night after working full-time as a security officer. It helped me build the skills that led to my first technology role as a Salesforce Administrator.

After sitting unused for over a year, the laptop is being repurposed as a Linux server for hands-on infrastructure learning.

## Decision

I will use the ThinkPad T530 as the foundation for a Cloud Operations Homelab.

The homelab will start as a headless Ubuntu Server and grow over time into a production-inspired environment that includes Linux administration, SSH, Bash automation, Docker, monitoring, Infrastructure as Code, CI/CD, Ansible, and cloud integration with AWS.

The project will be documented publicly on GitHub as both an infrastructure project and an engineering journal.

## Why This Matters

This project is designed to show practical engineering growth.

Instead of only showing completed labs or polished final results, the repository will document the full process:

- Planning
- Design decisions
- Implementation
- Mistakes
- Troubleshooting
- Lessons learned
- Improvements over time

The goal is to demonstrate how I think, build, troubleshoot, and improve as an engineer.

## Consequences

This project will require consistent documentation.

Technical decisions should be recorded as Architecture Decision Records when they affect the direction of the project.

Each major phase should include documentation that explains not only what was done, but why it was done.

This approach will take more time than simply building the lab, but it will create a stronger and more useful engineering portfolio.
